Benutzernamen, Passwörter und Gruppen: Zusätzlich

Die Verwaltung von Benutzernamen, Passwörter und Gruppen, zusammen mit den dazugehörigen Parametern, erfolgt über mehrere Reiter:

  • Basis: alle notwendigen Angaben für die Passwortverwaltung
  • Zusätzlich: weitere Angaben, inklusive Historie
  • Datum und Uhrzeit: Ablaufzeiten und Statistiken
  • Passwortrichtlinie: Abweichungen vom Standard

Reiter Zusätzlich

  • Autom. Eingabe: Wenn dieses Feld nicht leer ist, wird dessen Inhalt den standardmäßigen Inhalt für das 'automatische Eingeben' ersetzen (dies wird oben angezeigt, und stammt entweder vom Password Safe Standard oder vom Wert in der Datenbank der mit "Verwaltung → Optionen → Verschiedenes" angegeben wurde). Für mehr Information, siehe Hilfe zu "Autom. Eingabe".
  • Ausführen: Wenn dieses Feld nicht leer ist, wird dessen Inhalt bei der Aktion 'Ausführen' als Kommando dem Betriebssystem übergeben. Es können Variablen wie Anwendername, Passwort, usw. mit übergeben werden. Für mehr Information, siehe Hilfe zu "Ausführen".
  • Doppelklick-Aktion und Umsch Doppelklick-Aktion :
  • Dies erlaubt es festzulegen welche Aktion Password Safe ausführt, wenn Sie ein Doppelklick bzw. Umsch-Doppelklick auf diesem Eintrag ausführen. Standardmäßig wird Kopiere Passwort für Doppelklick , und Kopiere Benutzername für Umsch-Doppelklick vorbelegt. Die standardmäßigen Aktionen werden im Dialog Verwalten → Optionen → Verschiedenes festgelegt.
  • Passworthistorie: Es ist möglich Password Safe so zu konfigurieren, dass es sich die letzten paar Passwörter eines Eintrages merkt. Dies kann eingestellt werden beim Erstellen des Eintrages, oder später wenn Sie diesen Eintrag bearbeiten. Die jüngsten Passwörter eines Eintrages werden angezeigt. Die Anzahl der Passwörter die sich Password Safe merkt, kann eingestellt werden. Eine genauere Erklärung zur Passworthistorie finden Sie hier.
  • Tastaturkürzel: Tastaturkürzel können einen Eintrag zugeordnet werden, und erlauben so die direkte Auswahl eines Eintrages, unabhängig von der momentanen Anzeige im Fenster, nur mit den entsprechen Tastaturkürzel.

    Zusätzliche Hinweise zu Tastaturkürzel für Einträge:
    • Tastaturkürzel für Einträge werden abgespeichert als Teil des Datenbanksatzes zum Eintrag. Allerdings sind in Password Safe auch schon Schnelltasten für verschiedene Menüpunkte hinterlegt. Der Gültigkeitsbereich dieser Menü-Schnelltasten erstreckt sich über alle Datenbanken die ein Anwender auf einem PC geöffnet hat. Diese Menü-Schnelltasten können vom Anwender angepasst werden (Verwalten → Optionen, Schnelltasten). Sie können sich auch unterscheiden, abhängig von der ausgewählten Sprache z.B. Englisch, Deutsch, usw.
    • Darum tritt, obwohl eine Warnung ausgegeben wird wenn ein Tastaturkürzel ausgewählt wird das schon einem Menüpunkt zugeordnet wurde, kein Konflikt auf wenn Sie später die Menü-Schnelltaste anpassen, oder die Sprache wechseln. Gleichermaßen könnte ein Tastaturkürzel (noch) unbelegt sein, wenn es einem Eintrag zugewiesen wird, und später zu einem Konflikt führen wenn eine Menü-Schnelltaste definiert wird, oder die Sprache geändert wird.
    • Weil Tastaturkürzel zu einem Eintrag keine Änderungen in die Datenbank vornehmen können -anders als ein Menüpunkt-Schnelltaste, z.B. zum Löschen eines Eintrages- haben diese als Sicherheitsmaßnahme Vorrang vor Menüpunkt-Schnelltasten.
    • Tastaturkürzel zu einem Eintrag müssen entweder die Alt- oder Strg-Taste (mit oder ohne Umsch-Taste) enthalten. Wenn eine der beiden bei der Eingabe nicht dabei ist, werden alle Kombinationen von Alt, Strg und Umsch in dieser Reihenfolge (Alt; Alt+Umsch; Strg; Strg+Umsch, Alt+Strg; Alt+Strg+Umsch) ausprobiert, bis eine einzigartige Kombination gefunden wurde. Zum Beispiel, wenn der Anwender "Umsch + 6" auswählt, könnte das Tastaturkürzel zu "Alt + Umsch + 6" abgeändert werden. Wenn hingegen aber "Strg + 6" ausgewählt würde, wird es nicht geändert.
    • Wenn ein Tastaturkürzel schon zu einem Eintrag zugeordnet ist, wird es unter folgenden Umständen von einem Eintrag entfernt:
      • Wenn ein Eintrag importiert wird (via XML oder Text). Hinweis: Dies wird auch im Importbericht vermerkt.
      • Wenn ein Eintrag hinzugefügt wird während des Zusammenfügens von Datenbanken. Hinweis: Dies wird auch im Bericht vermerkt.
      • Wenn ein Eintrag eingetragen wird per "Drag & Drop" aus einer anderen Datenbank. Innerhalb der selben Datenbank ist dies nicht der Fall.
      • Wenn Einträge kopiert werden zur Datenbank als Ergebnis eines Vergleich-Dialoges.
      • Jeder Eintrag der mit Duplizier-Aktionen entstanden ist.

    Eine Liste der aktuell verwendeten Tastaturkürzel wird unter der Liste der Menu-Schnelltasten via "Verwalten → Optionen → Schnelltasten" angezeigt. Mit einem Doppelklick auf dem Tastaturkürzel kommt man zum Dialog Anzeigen/Bearbeiten des jeweiligen Eintrages. Ebenfalls kann man Tastaturkürzel als Merkmal in 'Filter Neu/bearbeiten' setzen um alle Einträge mit oder ohne Tastaturkürzel zu sehen.
