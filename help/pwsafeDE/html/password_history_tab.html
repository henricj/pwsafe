Passworthistorie



Im Reiter Passworthistorie werden die Regeln, wie die Historie der Einträge in Password Safe festgehalten wird, beschrieben.
Wenn Sie das Passwort eines Eintrages ändern, kann Password Safe die alten Passwörter, die zu diesem Eintrag gehören, abspeichern. Dies ist dann z.B. nützlich, wenn die Änderung des Passwortes nicht das erhoffte Ergebnis gebracht hat in der Anwendung, in dem das Passwort eingesetzt wurde. Auch kann festgelegt werden wie viele Passwörter für einen neuen Eintrag standardmäßig gespeichert werden sollen. Dies kann allerdings für einen bestimmten Eintrag wieder aufgehoben werden im Dialog "Eintrag bearbeiten" oder "Eintrag einfügen".

Speichere N bisherige Passwörter pro Eintrag

Wenn Sie dies markieren, und die Anzahl festlegen, definieren Sie hiermit den standardmäßigen Wert wie viele Passwörter festgehalten werden für jeden neuen Eintrag.

Verwalte Passworthistorie der aktuellen Einträge

Hiermit können Sie die Passworthistorie von bereits bestehenden Einträge abändern, wie folgt:
  • Keine Änderung - hiermit bleibt der bestehende Zustand erhalten, es ändert sich nichts.
  • Beende das Speichern vorheriger Passwörter - dies ist gleichbedeutend mit dem Löschen der Markierung zu "Speichere N bisherige Passwörter pro Eintrag" in allen Einträgen der Datenbank. Es geht hierbei keine Historie verloren, lediglich das Abspeichern der neuen Passwörter wird angehalten bis das Speichern der Passwörter wieder reaktiviert wird.
  • Beginne mit der Speicherung vorheriger Passwörter - dies ist gleichbedeutend mit dem Setzen der Markierung zu "Speichere N bisherige Passwörter pro Eintrag" in allen Einträgen der Datenbank. Einträge die bis dato keine Historie hatten, werden nun bearbeitet bis zur angegebenen Anzahl. Einträge die schon eine Historie hatten werden nur wieder bearbeitet, die Anzahl ändert sich aber nicht.
  • Setze die maximale Anzahl an gespeicherten Passwörtern auf obigen Wert - sämtliche Einträge werden auf die eingegebene Anzahl gesetzt. Hierbei geht keine Historie verloren, auch wenn ein Eintrag mehr als diese Anzahl Elemente gespeichert haben sollte.
